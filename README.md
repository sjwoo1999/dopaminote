# 🧠 Dopaminote

도파민 소비 패턴을 분석하고 개선하는 디지털 웰빙 웹 애플리케이션입니다.

## 📋 서비스 개요

**Dopaminote**는 사용자의 도파민 소비 패턴을 분석하고 개선하는 디지털 웰빙 웹 애플리케이션입니다.

### 🎯 핵심 가치 제안
> "무의식적인 도파민 소비를 의식적으로 바꿔서 건강한 디지털 라이프를 만들어보세요"

---

## 🎯 문제 정의

### 현재 상황
- 스마트폰 사용 시간 증가
- 소셜미디어 중독 현상
- 무의식적인 스크롤링과 콘텐츠 소비
- 디지털 디톡스의 필요성 증가

### 해결하고자 하는 문제
- 도파민 소비 패턴의 무의식적 반복
- 디지털 웰빙에 대한 인식 부족
- 개인별 맞춤형 개선 방안 부재

---

## 🚀 서비스 기능

### 1. 📸 도파민 기록 (`/record`)
**기능**: 스크린샷 업로드와 상황/기분 기록
- **이미지 업로드**: 파일 선택, 미리보기, 검증 (5MB 제한)
- **상황 선택**: 심심함, 스트레스, 습관, 소셜미디어, 업무, 엔터테인먼트, 기타
- **기분 기록**: 좋음, 무감정, 나쁨
- **웰빙 지표**: 사용 시간, 반복 횟수, 스트레스 수준
- **실시간 스코어**: 입력할 때마다 도파민 스코어 계산
- **추가 메모**: 상황에 대한 생각이나 느낌 기록

### 2. 🧠 웰빙 대시보드 (`/wellness`)
**기능**: 종합적인 디지털 웰빙 관리
- **실시간 도파민 스코어**: 현재 상태 시각화 및 트렌드 분석
- **Word to Self 계약**: 자기계약 시스템으로 행동 변화 유도
- **리셋 루틴**: 도파민 리셋 활동 관리 (명상, 호흡, 감사, 운동, 독서)
- **Integrity Circle**: 사회적 연대를 통한 동기부여 (준비 중)
- **개인화된 조언**: 데이터 기반 맞춤형 웰빙 가이드

### 3. 📊 패턴 분석 (`/report`)
**기능**: 기록된 데이터 분석 및 시각화
- **통계 대시보드**: 총 기록 수, 평균 기분, 주요 상황
- **차트 시각화**: 상황별/기분별 분포 차트
- **주간 트렌드**: 주별 도파민 스코어 변화 추이
- **개인화된 피드백**: 데이터 기반 개선 조언
- **패턴 발견**: 언제, 왜 도파민을 찾게 되는지 분석
- **자기 신뢰도 점수**: 계약 이행률 기반 신뢰도 측정

### 4. 📝 회고 노트 (`/journal`)
**기능**: 일일 회고 작성 및 목표 설정
- **일일 회고**: 도파민 소비 패턴 돌아보기
- **목표 설정**: 개선을 위한 구체적 목표 수립
- **감사 노트**: 긍정적인 변화에 대한 감사 기록
- **기분 트렌드**: 일별 기분 변화 추적
- **과거 기록 조회**: 회고 히스토리 확인
- **진행 상황 추적**: 목표 달성도 모니터링

```
목표: 건강한 디지털 라이프를 만들고 싶음
동기: 무의식적인 스크롤링을 줄이고 의미 있는 시간을 만들고 싶음
```

## 🛠️ 기술적 특징

### 기술 스택
- **Frontend**: Next.js 15, React 19, TypeScript
- **Styling**: Tailwind CSS 4
- **Database**: Supabase (PostgreSQL) - 설정됨, 미연결
- **Storage**: Supabase Storage (이미지 저장) - 준비됨
- **Charts**: Recharts
- **Icons**: Lucide React
- **Build**: Turbopack

### 아키텍처
- **App Router**: Next.js 15의 최신 라우팅 시스템
- **반응형 디자인**: 모바일/데스크톱 호환
- **타입 안정성**: TypeScript로 런타임 에러 방지
- **확장 가능성**: 컴포넌트 기반 모듈화 설계
- **상태 관리**: React hooks 기반 로컬 상태 관리

### 핵심 알고리즘
- **도파민 스코어 계산**: 시간(40점) + 반복패턴(30점) + 스트레스(30점) = 0-100점
- **실시간 분석**: 사용자 입력 시 즉시 스코어 업데이트
- **개인화 피드백**: 데이터 기반 맞춤형 조언 생성

## 🚀 시작하기

### 1. 저장소 클론

```bash
git clone <repository-url>
cd dopaminote
```

### 2. 의존성 설치

```bash
npm install
```

### 3. 환경 변수 설정

프로젝트 루트에 `.env.local` 파일을 생성하고 다음 내용을 추가하세요:

```env
# Supabase 설정
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

### 4. 개발 서버 실행

```bash
npm run dev
```

개발 서버가 [http://localhost:3001](http://localhost:3001)에서 실행됩니다.
